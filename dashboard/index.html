How it works
Set up once.
Protected forever.
01
Connect your repo
Click "Connect GitHub", pick your repo. Guardian auto-installs the webhook silently. No config files, no CLI, no YAML.
02
Open a pull request
Your normal workflow, unchanged. Push to a branch, open a PR. Guardian activates automatically the moment you do.
03
Agents go to work
Three AI agents run in sequence: find the vulnerability, patch it, verify the fix. No human needed in the loop.
04
Review & merge
Guardian posts a plain-English comment on your PR. See exactly what was found and how it was fixed. One click to approve.
The swarm
Three agents.
One mission.
A
The Hacker
Offense
Reads your diff, reasons about attack vectors, writes a proof-of-concept exploit and runs it. If the exploit succeeds, the vulnerability is real.
B
The Engineer
Defense
Receives the vulnerable code and the working exploit. Rewrites the function securely, preserving all logic and signatures. Minimal diff, production-quality fix.
C
The Reviewer
Verification
Runs the test suite and the exploit against the patch. If anything fails, sends error logs back to Agent B. Approves only when both tests pass and exploit fails.
